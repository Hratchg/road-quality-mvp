    ```python
    def _resolve_model_path(value: str | None) -> str:
        """Turn a YOLO_MODEL_PATH value into an on-disk .pt file path.

        Accepts:
            "user/repo"              -> hf_hub_download from that repo (best.pt by convention)
            "user/repo:filename.pt"  -> specific file inside the repo
            "user/repo@revision"     -> pinned revision (passed through to hf_hub_download)
            "/abs/path.pt"           -> used as-is
            "./relative/path.pt"     -> used as-is (existence checked lazily by YOLOv8Detector)
            None                     -> falls back to _DEFAULT_HF_REPO
        """
        target = value or _DEFAULT_HF_REPO

        # Explicit local path: starts with ./ or / or is an existing .pt file
        if target.startswith(("./", "/", "../")) or (target.endswith(".pt") and Path(target).exists()):
            return target

        # HF repo id (optional :filename suffix, optional @revision)
        repo_with_rev, _, filename = target.partition(":")
        repo_id, _, revision = repo_with_rev.partition("@")
        if not _HF_REPO_PATTERN.match(repo_with_rev):
            # Not a recognizable HF id; treat as local path, let YOLOv8Detector raise on load
            return target

        from huggingface_hub import hf_hub_download  # lazy import
        kwargs = {"repo_id": repo_id, "filename": filename or _DEFAULT_HF_FILENAME}
        if revision:
            kwargs["revision"] = revision
        return hf_hub_download(**kwargs)
    ```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Operator env → detector_factory | `YOLO_MODEL_PATH` value from shell env is read and interpreted as HF repo id or local path |
| Operator → huggingface.co | `hf_hub_download` fetches a `.pt` file (pickle payload) from a third-party CDN |
| detector_factory → YOLOv8Detector | Resolved local path is passed to `ultralytics.YOLO()` which deserializes pickle |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering + Elevation | `YOLO()` loading `.pt` file (pickle ACE) | mitigate | Default `YOLO_MODEL_PATH` points to a **known publisher** (`keremberke/yolov8s-pothole-segmentation`) whose repo is public and reviewed. Document in Plan 05's DETECTOR_EVAL.md "Security & Licensing" section. Recommend pinning `@revision` in production (see Pitfall 8 in 02-RESEARCH.md). This plan adds a comment in `.env.example` pointing operators to the risk. |
| T-02-02 | Information Disclosure | `HUGGINGFACE_TOKEN` leakage via logs | mitigate | `detector_factory.py` never logs env values verbatim — only the RESOLVED local path (which is a cache path, not the token). No `logger.info("token=...")` calls anywhere. `HUGGINGFACE_TOKEN` is consumed only in Plan 04's `finetune_detector.py --push` flow; factory never reads it. |
| T-02-03 | Tampering | Malicious `YOLO_MODEL_PATH` value exploiting regex matcher (e.g., path traversal `../../etc/passwd.pt`) | mitigate | `_resolve_model_path` treats values starting with `./`, `/`, or `../` as local paths and passes them through unvalidated — but consumer is `YOLOv8Detector._load_model` which uses `Path(self.model_path).exists()` and will raise FileNotFoundError if the path doesn't resolve to a real `.pt` file. No shell invocation, no `os.system`, no subprocess — so a traversal value is just a failed file open. Accepted risk: operator-controlled env var. |
| T-02-04 | Information Disclosure | `YOLO_MODEL_PATH` value logged via `logger.info("resolved_model_path=%s", resolved)` could reveal local filesystem structure in shared log aggregation | accept | Resolved paths are typically HF cache dirs (`~/.cache/huggingface/hub/models--...`) or local model paths. No secrets in paths. Documented logging behavior is the project convention (matches `backend/app/db.py` which logs nothing secret). |
| T-02-05 | Tampering | `hf_hub_download` MITM returning wrong `best.pt` | mitigate | `huggingface_hub` uses HTTPS + ETag/SHA256 for LFS files natively. Defense-in-depth via pinned `@revision` pattern supported by `_resolve_model_path`. Operator opts in by setting `YOLO_MODEL_PATH=user/repo@<commit_sha>`. |
| T-02-06 | Denial of Service | Arbitrary huge model file downloaded via `hf_hub_download` fills disk | accept | HF caches in `~/.cache/huggingface` with built-in dedup; operator opts in by setting env var. Not a public endpoint. |

# Plan 06-06: README + public demo announcement

## Goal

Update `README.md` to link the public demo URL with an honest data-source
+ accuracy explanation, then announce. This is the M1 Phase 6 closure
gate per CONTEXT D-08 — until this commit lands, the URL exists but
isn't promoted.

## Acceptance criteria

1. `README.md` includes a "## Live Demo" section with the public URL
   `https://road-quality-frontend.fly.dev/`
2. README clearly states the data source: "Real Mapillary CC-BY-SA
   street-level imagery from LA, processed through
   `keremberke/yolov8s-pothole-segmentation` (public baseline)"
3. README links to `docs/DETECTOR_EVAL.md` for accuracy numbers
4. README links to `docs/MAPILLARY_INGEST.md` for the ingestion pipeline
5. README has an explicit "demo, not production" disclaimer including:
   - "Numbers above measured on a 17-image / 3-positive LA test split;
     confidence intervals wide"
   - "LA-specific fine-tuning is **Phase 7** work; this demo uses the
     public baseline"
6. README includes a brief "What you can do" section: open the URL,
   sign in (or click "Try as demo"), pick two LA points, see
   fastest-vs-best route comparison
7. M1 Phase 6 verification artifacts (VERIFICATION.md, etc.) reflect
   the SCs being met
8. Commit message marks the announcement gate

## Tasks

1. **Read current README.md.** Identify where the demo section best fits
   (probably near the top, after a brief project description).

2. **Draft "## Live Demo" section** with:
   - Public URL prominent
   - 2-3 sentence "what is this" intro
   - Bullet list of "What you can do"
   - Subsection "Data + accuracy" with the public-baseline disclosure

3. **Draft "## Disclaimer" section** with the small-test-set caveat
   and the Phase 7 forward pointer.

4. **Update `## Quick start`** (or equivalent local-dev section) so
   it doesn't conflict with the new demo section. The local-dev
   workflow should still be documented; just clarify it's optional
   for users who only want to try the demo.

5. **Verify the live URL** works one more time:
   ```
   curl -s -o /dev/null -w 'HTTP %{http_code}\n' \
     https://road-quality-frontend.fly.dev/
   ```

6. **Commit** atomically:
   ```
   git add README.md
   git commit -m "feat(06-06): announce public demo — phase 6 closure"
   git push
   ```

7. **Optional: announce externally.** Tweet, blog post, share with
   colleagues, etc. Phase 6 is now complete regardless of whether
   this happens.

# Plan 06-03: Real Mapillary ingestion against prod DB

## Goal

Run `scripts/ingest_mapillary.py` against the deployed `road-quality-db`
to populate `segment_defects` with `source='mapillary'` rows. Phase 6
SC #2 ("Map View shows real Mapillary-ingested detections") becomes
literally true. Detector used: the public pretrained
`keremberke/yolov8s-pothole-segmentation` (default in
`data_pipeline/detector_factory.py`).

This will be re-run after Plan 06-06 (training + HF publish) to swap in
the trained-detector output.

## Acceptance criteria

1. `flyctl proxy 15432:5432 -a road-quality-db` succeeds + persists
   while the script runs.
2. `scripts/ingest_mapillary.py` exits 0.
3. `SELECT count(*) FROM segment_defects WHERE source='mapillary'`
   returns > 0 against prod.
4. `scripts/compute_scores.py --source mapillary` exits 0 +
   `segment_scores` reflects the new defect rows.
5. https://road-quality-frontend.fly.dev/ map view renders different
   colors than the synthetic-only baseline (visual smoke).
6. NO change to detector_factory or any committed code — pure data
   ingestion.

## Tasks

1. **Pre-flight checks:**
   - Verify ultralytics + torch + huggingface_hub installed in /tmp/rq-venv
     (done in 06-02 prep)
   - Verify MAPILLARY_ACCESS_TOKEN parseable from .env
   - Verify POSTGRES_PASSWORD parseable from .secrets-local/fly-prod.env

2. **Start flyctl proxy** in background:
   `flyctl proxy 15432:5432 -a road-quality-db &`
   (proxy is fine for short INSERT queries — Phase 5 anti-pattern is
   specifically about long DDL like pgr_createTopology, NOT inserts)

3. **Construct DATABASE_URL** for prod via localhost:15432:
   `DATABASE_URL=postgres://rq:<POSTGRES_PASSWORD>@localhost:15432/roadquality?sslmode=disable`

4. **Run ingest** with conservative bbox + image count:
   ```
   MAPILLARY_ACCESS_TOKEN=... \
   DATABASE_URL=... \
   /tmp/rq-venv/bin/python scripts/ingest_mapillary.py \
     --bbox -118.258,34.043,-118.248,34.053 \
     --max-images 50
   ```
   (small first run to validate end-to-end before pulling more)

5. **Run compute_scores** to refresh:
   ```
   DATABASE_URL=... \
   /tmp/rq-venv/bin/python scripts/compute_scores.py --source mapillary
   ```

6. **Verify** via psql proxy:
   - count of source='mapillary' defects > 0
   - count of segment_scores rows updated

7. **Visual smoke:** open https://road-quality-frontend.fly.dev/ and
   confirm Map View loads (no regression).

8. **Cleanup:** kill the flyctl proxy.

## Verification

```bash
# Defect rows from real Mapillary
flyctl ssh console -a road-quality-db -C \
  "psql -U rq -d roadquality -c \"SELECT count(*), min(detected_at), max(detected_at) FROM segment_defects WHERE source='mapillary';\""

# Segment scores updated
flyctl ssh console -a road-quality-db -C \
  "psql -U rq -d roadquality -c \"SELECT count(*) FROM segment_scores WHERE updated_at > NOW() - INTERVAL '1 hour';\""

# Frontend smoke
curl -s -o /dev/null -w 'HTTP %{http_code}\n' https://road-quality-frontend.fly.dev/
```

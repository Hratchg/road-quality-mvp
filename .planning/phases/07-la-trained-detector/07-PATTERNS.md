### `data_pipeline/eval.py` — add `metric="map50"` path to `bootstrap_ci`

**Analog:** `data_pipeline/eval.py:123-166` (existing `bootstrap_ci` implementation)

**Current signature at lines 123-129:**

```python
    per_image_counts: list[dict[str, int]],
    metric: Literal["precision", "recall"],
    n_resamples: int = 1000,
    ci_level: float = 0.95,
    seed: int = DEFAULT_SEED,
) -> tuple[float, float, float]:
```

**Current `_stat` inner function at lines 149-153:**
```python
def _stat(idxs: np.ndarray) -> float:
    tp, fp, fn = counts[idxs].sum(axis=0)
    if metric == "precision":
        return float(tp / (tp + fp)) if (tp + fp) > 0 else float("nan")
    return float(tp / (tp + fn)) if (tp + fn) > 0 else float("nan")
```

**Full bootstrap loop pattern at lines 155-166 (the resample engine to reuse for map50):**
```python
idxs = np.arange(len(counts))
rng = np.random.default_rng(seed)
samples = rng.choice(idxs, size=(n_resamples, len(idxs)), replace=True)
vals = np.array([_stat(s) for s in samples])
vals = vals[~np.isnan(vals)]
if len(vals) == 0:
    return (float("nan"), 0.0, float("nan"))
alpha = (1 - ci_level) / 2
low = float(np.percentile(vals, alpha * 100))
high = float(np.percentile(vals, (1 - alpha) * 100))
point = _stat(idxs)
return (low, point, high)
```

**Extension plan for `metric="map50"`:**
- New overload signature: accept `per_image_pairs: list[dict]` where each dict has `pred_boxes` and `gt_boxes` (list of tuples). The input type diverges from the TP/FP/FN-count input used for P/R.
- Recommended approach: add a separate `bootstrap_ci_map50` function (not overloading `bootstrap_ci`) to avoid breaking the existing Literal type. The new function pre-collects `(pred_boxes, gt_boxes)` per image, resamples image indices using the same `rng.choice` pattern (lines 157), and recomputes P-R curve + AUC per resample via `match_predictions` (lines 84-120).
- The `iou_xywh` (lines 63-81) and `match_predictions` (lines 84-120) helpers are already present — the new function only adds the P-R curve integration step (sort predictions by confidence across the resampled image set, compute cumulative TP/FP, derive P-R at each threshold step, integrate via trapezoid rule).
- Seed, n_resamples, ci_level parameters must match `bootstrap_ci` defaults (seed=42, n_resamples=1000, ci_level=0.95) per Phase 2 D-08.
